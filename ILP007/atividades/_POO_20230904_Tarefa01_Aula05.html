O que é Herança - POO

Herança dentro da perspectiva de POO, é a relação entre uma ou mais classes onde atributos e métodos podem ser herdados. Esta relação consiste em uma Superclasse que é a classe que contem os atributos e/ou métodos que serão herdados por subclasses (ou classes filhas). Desta forma classes filhas podem usufruir da mesma estrutura de atributos e métodos definidos na superclasse (classe mãe). Uma subclasse deve explicitar o uso da superclasse através do argumento extends + o nome da superclasse. A subclasse saberá de quem herdar as definições, no entanto a superclasse não sabe para quantas ou quais classes filhas transmite seus atributos e métodos. Uma subclasse eventualmente pode ser uma superclasse para outras classes definidas na estrutura, ou seja as classes que são filhas das classes filhas herdarão os atributos e métodos da superclasse imediatamente acima e consequentemente os atributos das classes que sua "mãe" herdou de outra superclasse (avó).

O que é Herança Múltipla

Herança múltipla é a possibilidade de uma classe filha (subclasse) herdar atributos e métodos de múltiplas superclasses. Em termos práticos é possível especializar uma classe com diferentes origens de métodos e atributos. No entanto esta abordagem pode ser complexa caso o planejamento das estruturas hierárquicas de classes não sejam bem definidas, pois pode ocorrer a redundância de nomes de atributos e métodos porém com propósitos muito diferentes.

Java: Suporta Herança Múltipla?

A linguagem Java não suporta herança múltipla direta. Isso significa que uma classe em Java não pode herdar diretamente de várias classes base afim de evitar problemas de ambiguidade e complexidade que podem surgir quando diferentes classes base têm membros com o mesmo nome. No entanto, Java oferece uma alternativa à herança múltipla por meio do conceito de interfaces. As interfaces em Java permitem que uma classe implemente múltiplas interfaces, adquirindo assim os comportamentos definidos por essas interfaces. Embora isso não seja exatamente herança múltipla, oferece uma maneira de alcançar funcionalidades semelhantes de uma maneira mais segura e controlada.

Linguagens com Herança Múltipla

Certas linguagens de programação suportam herança múltipla direta, permitindo que uma classe derivada herde de várias classes base. Algumas das linguagens que suportam esse recurso são C++, Python e Ruby.

Herança em C#

Supondo um sistema de gerenciamento de veículos, pode se definir uma superclasse chamada "Veiculo" com propriedades como "Marca", "Modelo" e métodos como "Ligar" e "Desligar". A partir disso pode se criar classes específicas para carros e motos, que herdarão características da classe base "Veiculo".


// Classe base
class Veiculo
{
public string Marca { get; set; }
public string Modelo { get; set; }

public void Ligar()
{
Console.WriteLine("O veículo está ligado.");
}

public void Desligar()
{
Console.WriteLine("O veículo está desligado.");
}
}

// Classe derivada para Carro
class Carro : Veiculo
{
public int NumeroPortas { get; set; }
}

// Classe derivada para Moto
class Moto : Veiculo
{
public bool PossuiPartidaEletrica { get; set; }
}

Neste exemplo, a classe "Carro" e a classe "Moto" herdam as propriedades e métodos da classe base "Veiculo". No entanto, cada uma delas também possui seus próprios membros específicos, como "NumeroPortas" para carros e "PossuiPartidaEletrica" para motos. A herança em C# facilita a reutilização de código e a criação de hierarquias de classes que representam relações do mundo real. Ela também permite que você substitua métodos da classe base usando o conceito de sobrescrita (override), dando flexibilidade para adaptar o comportamento das classes derivadas conforme necessário.
